# AI 协作课程素材与引用

## 1. 使用原则

本页素材优先选择官方或一手来源，适合：

- 放在 PPT 页脚作为引用
- 做讲课时的数据支撑
- 在 Q&A 环节回应“这个判断依据是什么”

建议使用方式：

- 一页 PPT 最多放 1-2 个关键数字
- 只引用能支撑观点的数字，不堆砌统计
- 厂商来源用于说明趋势，第三方来源用于增强可信度

## 3. 市场增长与采用情况

### 3.1 Stanford AI Index 2025

- 标题：`The 2025 AI Index Report`
- 链接：<https://hai.stanford.edu/ai-index/2025-ai-index-report>
- 可用于支撑：AI 已进入规模化采用阶段，且成本持续下降
- 可引用点：
  - `78%` 的组织在 2024 年报告已使用 AI，高于上一年的 `55%`
  - 2024 年全球生成式 AI 私人投资达到 `339 亿美元`
  - 达到 GPT-3.5 水平的推理成本在 2022-2024 年间下降 `280 倍以上`

建议讲法：

`AI 不是在变得更神秘，而是在变得更便宜、更普及、更能进入企业。`

### 3.2 Microsoft Work Trend Index 2025

- 标题：`2025: The year the Frontier Firm is born`
- 链接：<https://www.microsoft.com/en-us/worklab/work-trend-index/2025-the-year-the-frontier-firm-is-born>
- 可用于支撑：组织层面正在从 Copilot 使用过渡到人类与 Agent 混编
- 可引用点：
  - 调研覆盖 `31,000` 名工作者、`31` 个国家
  - `82%` 的领导者认为今年是重构战略和运营的关键年
  - `81%` 的领导者预计未来 12-18 个月 Agent 会中度或广泛纳入 AI 战略
  - `24%` 的领导者表示其组织已经在全公司范围部署 AI

建议讲法：

`组织真正关心的已经不是要不要用 AI，而是怎样把 AI 变成结构性能力。`

### 3.3 Microsoft 关于 Agent 准备度的研究

- 标题：`Agents are here-is your company prepared?`
- 链接：<https://www.microsoft.com/en-us/worklab/agents-are-here-is-your-company-prepared>
- 可用于支撑：Agent 落地的瓶颈更多在组织准备度，而不只在模型能力
- 可引用点：
  - 调研覆盖 `500` 位企业决策者、`13` 个国家、`16` 个行业
  - 准备度高的组织，预计扩展速度约为低准备度组织的 `2.5 倍`
  - 近 `80%` 组织认为跨团队数据共享不足以支撑 agentic AI

建议讲法：

`很多组织不是没有模型，而是没有把数据、流程和边界准备好。`

## 5. Agent 设计方法与反馈闭环

### 5.1 Anthropic - Building effective agents

- 标题：`Building effective agents`
- 链接：<https://www.anthropic.com/engineering/building-effective-agents>
- 可用于支撑：真正有效的 Agent 更依赖简单、可组合的工作流，而不是复杂框架
- 可引用点：
  - 文中强调成功落地更依赖 `simple, composable patterns`
  - 明确区分 `workflows` 和 `agents`
  - 推荐 `evaluator-optimizer` 回路，以及在关键点引入人工反馈

建议讲法：

`能跑起来的 Agent，通常不是最复杂的那个，而是上下文清楚、反馈明确、结构简单的那个。`

### 5.2 OpenAI - Compaction 文档

- 标题：`Compaction`
- 链接：<https://developers.openai.com/api/docs/guides/compaction>
- 可用于支撑：上下文工程的核心不是越多越好，而是保留高信号状态
- 可引用点：
  - 官方明确 compaction 目标是平衡 `quality, cost, and latency`
  - 上下文压缩后保留继续完成任务所需状态，而不是盲目堆上下文

建议讲法：

`上下文工程不是给模型喂更多内容，而是决定它应该记住什么、忽略什么。`

## 6. 你可以直接放进 PPT 的关键观点

下面这些句子可以直接做页面标题或结论：

- `AI 不是一个更聪明的工具，而是在重写工作的基本单元。`
- `真正的变化，不是 AI 会不会回答，而是它能不能进入工作流。`
- `LLM 解决的是“会不会想”，Agent 解决的是“能不能做完”。`
- `大多数人还停在会用 AI，真正的差距来自会不会设计协作系统。`
- `最被低估的一条：约束优于指令。`
- `步骤会过时，但约束能适配变化。`
- `能否把 AI 用进工作流，核心不是模型多强，而是上下文、接口和验证是否完善。`

## 7. 对外表达时的谨慎点

为了避免内容显得过度乐观，建议在课上同时讲清楚边界：

- Agent 仍然存在稳定性问题，不适合无限放权
- 企业引入 AI 的障碍，很多时候不是模型，而是数据、流程、权限和治理
- 并不是所有场景都适合全自动，很多场景仍然需要人类判断和最终责任
- Level 7-8 的后台 Agent 和 Agent 团队，目前仍然更接近前沿探索，而不是所有组织的生产默认

建议表达：

`我们既不要低估 AI，也不要高估 AI。真正有价值的是找到它今天能稳定创造价值的位置。`
